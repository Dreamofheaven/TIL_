<SQL문 유형>
목적에 따라서 4가지 범주로 나뉨 (명확하게 나뉜게 아님, 관점에 따라 달라)
SQL은 데이터를 조작하려고 쓰는거, 소통을 하려고 쓰는 것이기에 

- DDL : 데이터 정의(데이터 기본 구조 및 형식 변경) : CREATE, DROP, ALTER
- DQL : 데이터 검색(데이터 검색) : SELECT
- DML : 데이터 조작(추가 수정 삭제) : INSERT, UPDATE, DELETE
- DCL : 데이터 제어(데이터 및 작업에 대한 사용자 권한 제어) COMMIT, ROLLBACK, GRANT, REVOKE

DB에서 제일 중요한 CRUD, 검색은 Read / 조작은 C, U, D / 정의는 구조에 대한 정의
데이터가 검색되고 조작하기 위해서는 구조가 있어야하니까, 구조가 먼저 정의되어야 CRUD를 할 수 있음
제어는 컨트롤의 영역이다. 사용자의 권한이나 계정을 컨트롤한다. 즉, 테이블을 직접 조작하는 것과는 멀다. 

DB의 실력은 어떻게 잘 검색해서 내가 원하는 데이터를 가져올 것인가? 이것이 중요하다.
그래서 DQL의 비중이 많다.

1) quering data (기본)
- SELECT 문의 역할 : 테이블에서 데이터를 조회 
- SELECT syntax 

```SQL
SELECT 
 select_list 
FROM 
 table_name; //(테이블이름 쓰기)이 테이블에서 조회를 할거야
```

- SELECT 키워드 다음에 데이터를 선택하려는 필드(열)를 하나 이상 지정
- FROM 키워드 다음에 데이터를 선택하려는 테이블의 이름 지정

<SELECT EX #1>

2번째는 커서표시된거만 실행
